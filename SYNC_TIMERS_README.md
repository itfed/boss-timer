# Скрипты синхронизации таймеров боссов

## Описание

Эти скрипты позволяют экспортировать время убийства боссов с сайта и синхронизировать локальные таймеры.

## Скрипты

### 1. export_timers.py

Экспортирует время убийства боссов с сайта в JSON формате.

### 2. update_local_timers.py

Обновляет локальные таймеры на основе экспортированных данных.

## Установка зависимостей

```bash
pip install requests pytz
```

## Использование

### 1. Экспорт таймеров с сайта:

```bash
python export_timers.py
```

Скрипт:
- Подключается к сайту и получает время убийства всех боссов
- Выводит информацию в консоль
- Сохраняет данные в файл `exported_timers_YYYYMMDD_HHMMSS.json`
- Показывает данные в формате для копирования

### 2. Обновление локальных таймеров:

```bash
python update_local_timers.py
```

Для обновления из файла экспорта:

```python
from update_local_timers import update_local_timers_from_file

# Обновить из файла экспорта
update_local_timers_from_file("exported_timers_20260112_020744.json", "http://localhost:5111")
```

Или используйте встроенную функцию:

```bash
python update_local_timers.py
```

И затем вручную укажите путь к файлу в коде.

## Настройка URL

По умолчанию скрипты используют:
- `http://localhost:5111` для локального сайта
- Можно изменить в коде скриптов

## Пример использования

### Полный цикл синхронизации:

1. Экспортируем с сайта:
```bash
python export_timers.py
```

2. Обновляем локальные таймеры:
```bash
# Вручную укажите имя файла в скрипте
```

Или программно:

## Формат данных

Файл экспорта содержит JSON в формате:
```json
{
  "1": "16:35:00",
  "2": "17:25:26", 
  "7": null,
  "8": null,
  ...
}
```

Где:
- Ключ - ID босса
- Значение - время убийства в формате "HH:MM:SS" или `null`, если босс не убит

## Особенности

- Все времена обрабатываются в московской таймзоне (UTC+3)
- Скрипт автоматически корректирует время, если оно в будущем
- Обрабатывает как убитых, так и неубитых боссов
- Показывает прогресс и результаты операций
- Содержит обработку ошибок

## Часовые пояса

ВАЖНО: Приложение использует московское время (UTC+3) для всех расчетов. Если вы находитесь в другом часовом поясе (например, Сахалин, UTC+11), могут возникать проблемы с синхронизацией из-за разницы в датах.

Для решения этой проблемы добавлен скрипт `timezone_aware_sync.py`, который автоматически учитывает разницу во времени между вашим местоположением и Москвой.

При разнице во времени:
- Когда в Москве 23:00 11 января, у вас может быть 07:00 12 января
- Время убийства босса в 22:00 в Москве будет восприниматься как "в будущем" относительно вашей текущей даты
- Новый скрипт корректно обрабатывает такие ситуации

## API Endpoints

Скрипты используют:
- `/get_boss_timers` - для получения информации о боссах
- `/manual_edit_time/<boss_id>` - для обновления времени убийства
